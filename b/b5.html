1. Dağıtık Sistemler (Distributed Systems)

Dağıtık Sistem, ortak bir hedefe ulaşmak için bir bilgisayar ağı üzerinden iletişim kuran ve birbirleriyle etkileşime giren birden fazla bilgisayardan oluşan bir sistemdir.

Dağıtık Sistemlerin 8 Yanılgısı (Fallacies)

Dağıtık sistem tasarlarken, geliştiricilerin sıklıkla düştüğü 8 yaygın yanlış varsayım (yanılgı) vardır:

Ağ güvenilirdir.
Gecikme (latency) sıfırdır.
Bant genişliği (bandwidth) sonsuzdur.
Ağ güvenlidir.
Topoloji (ağ yapısı) değişmez.
Tek bir yönetici vardır.
Taşıma maliyeti sıfırdır.
Ağ homojendir.

Bu varsayımların tümü yanlıştır ve sistem tasarımında dikkate alınmalıdır.

Dağıtık Sistemlerin Temel Özellikleri ve Zorlukları

Temel Özellikler

  • Ölçeklenebilirlik (Scalability): Yeni düğümler ekleyerek sistemin büyüme yeteneği.
  • Eşzamanlılık (Concurrency): Bileşenlerin aynı anda çalışması (global bir saat yoktur).
  • Hata Toleransı (Fault Tolerance): Bir düğüm arızalansa bile sistemin çalışmaya devam etmesi.
  • Şeffaflık (Transparency): Sistemin dışarıdan tek bir birim olarak görünmesi.
  • Heterojenlik (Heterogeneity): Farklı donanım ve yazılımların bir arada çalışması.
  • Çoğaltma (Replication): Güvenilirliği artırmak için bileşenlerin kopyalanması.

Zorluklar

  • Artan Hata Olasılığı: Ne kadar çok bileşen varsa, hata olasılığı o kadar artar.
  • Senkronizasyon Zorlukları: Global bir saat olmadığı için yarış koşulları (race conditions) ve kilitlenme (deadlock) riski.
  • Mükemmel Olmayan Ölçeklenebilirlik: Düğüm sayısını ikiye katlamak performansı ikiye katlamaz.
  • Artan Güvenlik Karmaşıklığı: Daha geniş bir saldırı yüzeyi (attack surface) oluşturur.

2. Zookeeper (Dağıtık Koordinasyon Servisi)

Zookeeper: Dağıtık uygulamaların birbiriyle koordine olmasını sağlayan açık kaynaklı bir servistir. Dağıtık mimaride yapılandırma bilgilerini, isimlendirmeyi korumak ve senkronizasyonu sağlamak için tasarlanmıştır.

Temel motivasyonu, yarış koşulları (race conditions) ve kilitlenme (deadlock) gibi hatalara yatkın olan koordinasyon servislerini sıfırdan uygulama zorluğunu ortadan kaldırmaktır.

Zookeeper Kullanım Alanları (Use Cases)
Lider Seçimi (Leader Election)

Dağıtık sistemde görevleri yönetecek bir "lider" (master) düğüm belirler. (Örn: Kafka broker'ları veya HDFS NameNode HA arasında lider seçimi).

Dağıtık Kilit (Distributed Locking)

Aynı kaynağa birden fazla düğümün aynı anda müdahale etmesini engelleyerek veri tutarlılığını sağlar.

Yapılandırma Yönetimi (Configuration Management)

Uygulama ayarlarının merkezi bir noktada saklanmasını sağlar. "Watch" mekanizması sayesinde, konfigürasyon değiştiğinde tüm sistemler anında haberdar olur.

Servis Keşfi (Service Discovery)

Dağıtık sistemde çalışan servislerin (IP adresleri değişse bile) birbirini bulmasını sağlar.

Koordinasyon ve Senkronizasyon

Farklı düğümlerin belirli bir sırayla işlem yapmasını sağlar.

Durum Takibi / Meta Veri Yönetimi

Sistem bileşenlerinin çalışma durumunu izler. Canlı olan ve çöken düğümleri bilir (Örn: Kafka, topic/partition bilgilerini Zookeeper'da tutar).

3 & 4. NoSQL ve MongoDB Mimarisi

Veri hacmindeki ve çeşitliliğindeki artış, geleneksel RDBMS sistemlerini yetersiz hale getirmiştir. NoSQL, bu soruna yatay ölçeklenebilen ve dinamik şemalı bir alternatif sunar.

Özellik RDBMS (İlişkisel, örn: MySQL) MongoDB (NoSQL)
Veri Modeli İlişkisel, Satır tabanlı (Row-based) Belge yönelimli (Document-based), JSON/BSON
Ölçeklenme Dikey Ölçeklenme (Vertical Scaling - RAM artırmak) Yatay Ölçeklenme (Horizontal Scaling - Sunucu eklemek)
Şema (Schema) Önceden Tanımlı Şema (Predefined Schema) Dinamik Şema (Dynamic Schema)
Temel Teorem ACID (Atomicity, Consistency, Isolation, Durability) CAP (Consistency, Availability, Partition tolerance)
Join (Birleştirme) Karmaşık JOIN işlemlerini destekler. JOIN desteği yoktur (Yerine Embedded Documents kullanılır).

Sharding, veriyi birden fazla makineye (sunucuya) dağıtma yöntemidir. MongoDB, çok büyük veri setlerini ve yüksek işlem hacmini desteklemek için sharding (yatay ölçeklenme) kullanır.

MongoDB Sharded Cluster Mimarisi

Mongos (Router - Yönlendirici)

İstemci isteklerini alır ve doğru Shard'a yönlendirir.

Config Servers (Replica Set)

Kümenin meta verilerini (hangi verinin hangi shard'da olduğu) saklar.

Shard A (Replica Set)

Verinin bir alt kümesini tutar.

Shard B (Replica Set)

Verinin başka bir alt kümesini tutar.

5, 6 & 7: MongoDB Kurulumu, CRUD ve ASP.NET Core

MongoDB Kurulumu ve CRUD Operasyonları

Adımlar:

Sunucuyu manuel yapılandırma ve başlatma adımları:

  1. MongoDB Community Server .msi indirilir ve kurulur (Compass ile birlikte).
  2. C:\mongodb\data klasörleri oluşturulur.
  3. C:\mongodb\mongo.config dosyası oluşturulur ve içine yollar eklenir.
  4. C:\mongodb\mongo.log adında boş bir log dosyası oluşturulur.
  5. CMD (Komut İstemi) açılır ve mongod.exe (sunucu) config dosyası ile başlatılır.

mongo.config Dosyası


# Veri dosyalarının yolu
dbpath = C:\mongodb\data
# Log dosyasının yolu
logpath = C:\mongodb\mongo.log

Sunucuyu Başlatma (CMD)


# MongoDB'nin bin klasörüne git
cd "C:\Program Files\MongoDB\Server\5.0\bin"

# Sunucuyu yapılandırma dosyasıyla başlat
mongod.exe --config C:\mongodb\mongo.config

İkinci bir CMD açılır ve mongo.exe (veya mongosh) komutu ile Mongo Shell başlatılır.


// 'firstexample' adında bir veritabanı oluşturur ve ona geçer
use firstexample;

// Mevcut veritabanındaki koleksiyonları (tabloları) listeler
show collections;

// CREATE: "person" adında yeni bir koleksiyon oluşturur
db.createCollection("person");

// CREATE: "person" koleksiyonuna yeni bir belge (kayıt) ekler
db.person.insert({adi: "Stephen Hawking"});
db.person.insert({adi: "Isaac Newton"});

// READ (All): "person" koleksiyonundaki tüm belgeleri listeler
db.person.find();
// Çıktı: { _id: ObjectId(...), adi: "Stephen Hawking" } ...

// READ (Filter): Adı "Isaac Newton" olanları bulur
db.person.find( {adi: "Isaac Newton"} );

Robo 3T (eski adıyla Robomongo), MongoDB için bir grafiksel kullanıcı arayüzüdür (GUI). Sunucuya (localhost:27017) bağlanarak veritabanlarını, koleksiyonları ve belgeleri görsel olarak yönetmenizi sağlar.

Bir koleksiyona çift tıklamak, db.getCollection('person').find({}) komutunu otomatik çalıştırır ve sonuçları bir tablo olarak gösterir. Ayrıca, komutları manuel olarak yazıp çalıştırabileceğiniz bir shell arayüzü de sunar.

ASP.NET Core ve MongoDB Uygulaması (CarGalleryApp)

MongoDB'nin bir ASP.NET Core MVC web uygulamasında nasıl veritabanı olarak kullanılacağına dair adımlar.

1. Proje Kurulumu

Visual Studio'da "ASP.NET Core Web Application" (Model-View-Controller) şablonu kullanılarak yeni bir proje (CarGalleryApp) oluşturulur.

2. MongoDB Driver Ekleme

Proje üzerinde "NuGet Package Manager" (NuGet Paket Yöneticisi) açılır. "Browse" sekmesinden MongoDB.Driver paketi aranır ve projeye yüklenir.

MongoDB'deki "Cars" koleksiyonunda saklanacak veriyi temsil eden bir C# sınıfı (Models/Cars.cs) oluşturulur.


using MongoDB.Bson;
using MongoDB.Bson.Serialization.Attributes;
using System.ComponentModel.DataAnnotations;

public class Cars
{
    [BsonId] // Bu alanın MongoDB _id'si olduğunu belirtir.
    [BsonRepresentation(BsonType.ObjectId)] // Tipi ObjectId olarak ayarlar.
    public string ID { get; set; }

    [Required] // ASP.NET doğrulaması
    [BsonElement("Model")] // MongoDB'deki alan adını belirtir.
    public string Model { get; set; }

    [Required]
    [BsonElement("Year")]
    public int Year { get; set; }

    [BsonElement("Price")]
    public decimal Price { get; set; }

    [BsonElement("ImageURL")]
    [Required]
    public string ImageURL { get; set; }
}

Veritabanı işlemlerini (CRUD) yönetecek bir servis sınıfı (CarServices.cs) oluşturulur.


using MongoDB.Driver;
using System.Collections.Generic;
using Microsoft.Extensions.Configuration;

public class CarServices
{
    private readonly IMongoCollection _cars;

    // Constructor: Veritabanı bağlantısını ayarlar
    public CarServices(IConfiguration config)
    {
        // Bağlantı dizesini appsettings.json'dan okur
        MongoClient client = new MongoClient(config.GetConnectionString("CarGalleryDb"));
        // "CarGalleryDb" veritabanını alır
        IMongoDatabase database = client.GetDatabase("CarGalleryDb");
        // "Cars" koleksiyonunu alır
        _cars = database.GetCollection("Cars");
    }

    // CREATE
    public Cars Create(Cars car)
    {
        _cars.InsertOne(car);
        return car;
    }

    // READ (Tümü)
    public List Get()
    {
        return _cars.Find(car => true).ToList();
    }

    // READ (Tek)
    public Cars Get(string id)
    {
        return _cars.Find(car => car.ID == id).FirstOrDefault();
    }

    // UPDATE
    public void Update(string id, Cars carIn)
    {
        _cars.ReplaceOne(car => car.ID == id, carIn);
    }

    // DELETE
    public void Remove(string id)
    {
        _cars.DeleteOne(car => car.ID == id);
    }
}

6. Servisi Kaydetme (Startup.cs)

CarServices sınıfı Dependency Injection'a eklenir.


public void ConfigureServices(IServiceCollection services)
{
    // CarServices sınıfını Scoped olarak ekle
    services.AddScoped();
    services.AddControllersWithViews();
}

Kullanıcıdan gelen web isteklerini işleyecek ve CarServices'i kullanacak Controller sınıfı.


using Microsoft.AspNetCore.Mvc;

public class CarsController : Controller
{
    private readonly CarServices _carService;

    // Servis, constructor injection ile otomatik olarak alınır
    public CarsController(CarServices carService)
    {
        _carService = carService;
    }

    // GET: CarsController (Liste sayfası)
    public ActionResult Index()
    {
        // Tüm arabaları servisten al ve View'a gönder
        return View(_carService.Get());
    }
    
    // POST: CarsController/Create (Yeni kayıt)
    [HttpPost]
    [ValidateAntiForgeryToken]
    public ActionResult Create(Cars car)
    {
        if (ModelState.IsValid)
        {
            _carService.Create(car);
            return RedirectToAction(nameof(Index));
        }
        return View(car);
    }

    // GET: CarsController/Edit/5 (Düzenleme sayfası)
    public ActionResult Edit(string id)
    {
        var car = _carService.Get(id);
        //... (hata kontrolü) ...
        return View(car);
    }

    // POST: CarsController/Edit/5 (Güncelleme)
    [HttpPost]
    [ValidateAntiForgeryToken]
    public ActionResult Edit(string id, Cars cars)
    {
        if (ModelState.IsValid)
        {
            _carService.Update(id, cars);
            return RedirectToAction(nameof(Index));
        }
        return View(cars);
    }
    
    // POST: CarsController/Delete/5 (Silme)
    [HttpPost, ActionName("Delete")]
    [ValidateAntiForgeryToken]
    public ActionResult DeleteConfirmed(string id)
    {
        var car = _carService.Get(id);
        //... (hata kontrolü) ...
        _carService.Remove(car.ID);
        return RedirectToAction(nameof(Index));
    }
}
