1. PCAP ve Libpcap Kavramı

PCAP, Packet Capture (Paket Yakalama) kelimelerinin kısaltmasıdır. Temel olarak ağ aktivitelerinin tüm kaydını tutan bir dosyadır. Bu kayıt, OSI modelinin Katman 2'den (Data-Link) Katman 7'ye (Uygulama) kadar olan verilerini içerebilir.

En Yaygın Kullanım: libpcap

Paket yakalama (PCAP) işlemini gerçekleştirmek için kullanılan ana kütüphane libpcap'tir.

  • Açık kaynaklıdır (open-source).
  • Hem *nix (Linux, macOS vb.) hem de Windows (WinPcap) sistemleriyle uyumludur.
  • Bir C kütüphanesidir ve C'nin yanı sıra Python, Java, C# gibi birçok dile de uyumluluk sağlar.

Paket Yakalama Akışı

Ağ trafiğini yakalama işlemi genellikle bir "Tap" veya "Inline Device" (Satır İçi Cihaz) aracılığıyla yapılır. İnternetten veya yerel ağdan gelen paketler, bu cihazlar tarafından kopyalanır ve analiz için Wireshark veya tcpdump gibi araçları çalıştıran bir bilgisayara yönlendirilir.

PCAP Kullanım Alanları

Ağ trafiğini dinlemek ve ağda neler olup bittiğini görmek.
Bant genişliği kullanımını kontrol etmek (Hangi uygulama ne kadar tüketiyor?).
Pasif DNS çözümlemesi yapmak (Hangi cihaz hangi domaine erişiyor?).
Saldırı girişimlerini ve anormal aktiviteleri tespit etmek.
Araştırma çalışmaları için ağ protokollerinin davranışını test etmek.

3 & 4. Ağ Dinleme Araçları (Sniffer) ve Wireshark

Sniffer Nedir ve Modları Nelerdir?

Sniffer (Koklayıcı) olarak da bilinen ağ dinleme araçları, ağ üzerindeki trafiği sezebilen, çözebilen ve hatta değiştirebilen donanım/yazılım araçlarıdır. Genellikle yazılım tabanlıdırlar (Örn: Wireshark, Windump).

Çalışma Prensibi: Makinenizdeki ağ kartından geçen (gönderilen veya alınan) paketlerin bir kopyasını alırlar.

Pasif İzleme (Sezme)

Sadece ağı dinler ve veri toplar. Ağa müdahale etmez.

Aktif (Atak Yapma)

Ağa müdahale eder, paketleri değiştirir veya sahte paketler oluşturur.

Sniffer Kullanıcıları (İyi vs. Kötü Niyetli)

1. Sistem Yöneticileri (İyi Niyetli)

  • Sorun Giderme: Sistem problemlerini ve ağ performansını anlamak.
  • Güvenlik: Saldırıları ve anormal trafiği tespit etmek.
  • Test: Uygulama operasyonlarını ve protokol davranışlarını test etmek.

2. Saldırganlar (Kötü Niyetli)

  • Veri Hırsızlığı: Şifrelenmemiş protokolleri (FTP, HTTP, POP3) dinlemek, VoIP konuşmalarını kaydetmek.
  • Keşif: Hedef ağın trafiğini ve desenini keşfetmek.
  • Sızma: Backdoor (arka kapı) teknikleri ile ağa dahil olmak.
Wireshark Detayları

Eski adı Ethereal olan (2006'da Wireshark oldu) en popüler paket sniffer uygulamasıdır. Fonksiyonelliği tcpdump'a benzer.

  • Hem GUI (Grafik Arayüz) hem de komut satırı özelliklerine sahiptir.
  • Paketleri sıralama ve filtreleme konusunda çok güçlüdür.
  • 750'nin üzerinde protokolü destekler ve bu protokollerin kapsüllenmiş yapısını yorumlar.
  • Pasif bir izleme aracıdır, kendisi ağ verisi üretmez.
  • Veri yakalamak için pcap (veya WinPcap) kütüphanesine ihtiyaç duyar.

Sniffer'ın Topolojideki Yeri

Bir sniffer'ın ağda nereye konumlandırıldığı, hangi trafiği görebileceğini belirler.

Hub'a Bağlı Sniffer

Hub (Katman 1 cihazı) "aptal"dır; gelen veriyi tüm portlara kopyalar. Bu nedenle, Hub'a bağlı bir sniffer, o Hub'daki tüm trafiği görebilir.

Switch'e Bağlı Sniffer

Switch (Katman 2 cihazı) "akıllı"dır; veriyi sadece hedef MAC adresinin bağlı olduğu port'a yönlendirir. Sniffer, özel bir yapılandırma (port mirroring) olmadan sadece kendi trafiğini ve broadcast trafiğini görür.

6. Wireshark Kullanıcı Arayüzü

1. Paket Listesi (Üst Bölüm)

Yakalanan paketlerin listesidir. Her satır bir pakettir.

  • No: Paket sırası
  • Time: Yakalanma zamanı
  • Source: Kaynak IP/MAC adresi
  • Destination: Hedef IP/MAC adresi
  • Protocol: Protokol tipi (örn: TCP, DNS, HTTP, ARP)
  • Info: Paket hakkında özet bilgi (örn: [SYN, ACK], Standard query)
2. Paket Detayları (Orta Bölüm)

Listeden seçilen paketin katmanlı yapısını ("kapsüllenmiş" halini) gösterir. Paketi katman katman "açabilirsiniz":

  • Frame (Fiziksel katman bilgisi)
  • Ethernet II (Data-Link Katmanı, MAC Header)
  • Internet Protocol (IP) (Ağ Katmanı, IP Header)
  • Transmission Control Protocol (TCP) (Taşıma Katmanı, TCP Header)
  • Hypertext Transfer Protocol (HTTP) (Uygulama Katmanı, Asıl Veri)
3. Paket Verisi (Alt Bölüm)

Seçilen paketin ham verisini iki formatta gösterir:

  • Sol Taraf (Hexadecimal): Paketin ham byte (onaltılık) gösterimi.
  • Sağ Taraf (ASCII): Bu ham byte'ların okunabilir metin (ASCII) karşılığı.

7. Libpcap Syntax Yapısı (C Kod Örnekleri)

Ağ dinlemesi yapacak varsayılan cihazı (network interface) pcap_lookupdev fonksiyonu kullanarak bulma.

#include <stdio.h>
#include <pcap.h>

int main(int argc, char *argv[]) {
    char *dev, errbuf[PCAP_ERRBUF_SIZE];

    // Varsayılan cihazı bul
    dev = pcap_lookupdev(errbuf);

    if (dev == NULL) {
        fprintf(stderr, "Couldn't find default device: %s\n", errbuf);
        return(2);
    }

    // Bulunan cihazın adını yazdır
    printf("Device: %s\n", dev);
    return(0);
}

Bir cihazı dinlemeye başlamak için pcap_open_live fonksiyonu kullanılır. Bu fonksiyon, bir dinleme oturumu (handle) döndürür.

#include <pcap.h>

pcap_t *handle; // Dinleme oturumu (session handle)
char *dev;       // Cihaz adı (örn: "eth0")
char errbuf[PCAP_ERRBUF_SIZE];

// dev'in pcap_lookupdev ile bulunduğunu varsayalım...

// Cihazı dinlemek için aç
// (device, buffer_size, promiscuous_mode, timeout_ms, error_buffer)
handle = pcap_open_live(dev, BUFSIZ, 1, 1000, errbuf);

if (handle == NULL) {
    fprintf(stderr, "Couldn't open device %s: %s\n", dev, errbuf);
    return(2);
}

Sadece belirli paketleri (örn: "port 23") yakalamak için filtreler kullanılır. Filtre derlenmeli (pcap_compile) ve uygulanmalıdır (pcap_setfilter).

#include <pcap.h>

pcap_t *handle;                  /* Session handle */
char filter_exp[] = "port 23"; /* Filtre ifadesi (Telnet) */
struct bpf_program fp;        /* Derlenmiş filtre */
bpf_u_int32 mask;              /* Ağ maskesi */
bpf_u_int32 net;               /* Ağ IP'si */
char *dev; // ... (dev ve handle'ın alındığını varsayalım)

// Cihazın ağ bilgilerini al
if (pcap_lookupnet(dev, &net, &mask, errbuf) == -1) {
    fprintf(stderr, "Can't get netmask for device %s\n", dev);
    net = 0; mask = 0;
}

// Filtre ifadesini derle (string'den BPF formatına)
if (pcap_compile(handle, &fp, filter_exp, 0, net) == -1) {
    fprintf(stderr, "Couldn't parse filter %s: %s\n", 
            filter_exp, pcap_geterr(handle));
    return(2);
}

// Derlenen filtreyi oturuma uygula
if (pcap_setfilter(handle, &fp) == -1) {
    fprintf(stderr, "Couldn't install filter %s: %s\n", 
            filter_exp, pcap_geterr(handle));
    return(2);
}
