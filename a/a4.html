1. IP Protokolü (IPv4)

Internet Protokolü (IP), TCP/IP protokol takımında Ağ Katmanı (Network Layer) tarafından kullanılan temel iletim mekanizmasıdır. Verinin bir ağdan diğerine yönlendirilmesinden sorumludur.

  • Datagram: IP protokolünde taşınan veri paketlerine datagram denir.
  • Başlık Boyutu: Bir IP datagram başlığı 20 ila 60 bayt arasında değişir.
  • Bağlantısız (Connectionless): Veri göndermeden önce alıcıyla bir bağlantı veya oturum kurmaz. Her paket birbirinden bağımsızdır.
  • En İyi Teslim (Best Effort): IP, paketin hedefe ulaşacağını garanti etmez. Güvenilirlik, üst katman (örn: TCP) tarafından sağlanır.
  • Ortam Bağımsız: Alttaki fiziksel teknolojiden (Ethernet, Wi-Fi, Fiber) bağımsız çalışır.
  • Kapsülleme (Encapsulation): IP datagramları, Katman 2'de bir çerçeve (frame) içine kapsüllenerek gönderilir.

IPv4 Başlık Yapısı (Min 20 Byte)

Başlık, paketin nasıl yönlendirileceği hakkında kritik bilgiler içerir. Her satır 32 bit (4 byte) uzunluğundadır.

2. IP Datagramı Fragmantasyonu (Parçalama)

Bir IP datagramı, farklı MTU (Maximum Transmission Unit) değerlerine sahip ağlardan geçerken parçalanabilir. Bir router, bir datagramı MTU'su daha küçük bir ağa göndermek zorundaysa ve paketin DF (Don't Fragment) bayrağı 0 ise, datagramı parçalamak zorundadır. Parçalar hedefte birleştirilir.

Detaylı Fragmantasyon Örneği (Yeniden Parçalama)

Orijinal Datagram ID: 14567, MF: 0, DF: 0 Total Length: 4020 (20B Başlık + 4000B Veri)
→
Fragment 1 (Ağ 1) ID: 14567, MF: 1, Offset: 0 Total Length: 1420 (20B Başlık + 1400B Veri [0-1399])
Fragment 2 (Ağ 1) ID: 14567, MF: 1, Offset: 175 (1400/8) Total Length: 1420 (20B Başlık + 1400B Veri [1400-2799])
Fragment 3 (Ağ 1) ID: 14567, MF: 0, Offset: 350 (2800/8) Total Length: 1220 (20B Başlık + 1200B Veri [2800-3999])
→
Fragment 2.1 (Ağ 2) ID: 14567, MF: 1, Offset: 175 Total Length: 820 (20B Başlık + 800B Veri [1400-2199])
Fragment 2.2 (Ağ 2) ID: 14567, MF: 1, Offset: 275 (2200/8) Total Length: 620 (20B Başlık + 600B Veri [2200-2799])

3. IP Opsiyonları ve Kontrol Toplamı (Checksum)

IP Datagramı Başlık Yapısı (Opsiyonlar)

IP başlığının değişken kısmı (maksimum 40 bayt) opsiyonel bilgiler içerir. Bu seçenekler genellikle ağ testi ve hata ayıklama için kullanılır. Her opsiyonun Type, Length ve Value alanları vardır.

Tek Baytlık Opsiyonlar

  • No Operation (NO-OP): Hizalama (padding) için kullanılır.
  • End of Option (END-OP): Opsiyon listesinin sonunu belirtir.

Çok Baytlık Opsiyonlar

  • Record Route: Paketin geçtiği router'ların IP'lerini kaydeder.
  • Strict Source Route: Paketin izlemesi gereken *kesin* rotayı belirler.
  • Loose Source Route: Paketin uğraması gereken router listesini belirler (aralara başka router girebilir).
  • Timestamp: Paketin router'lardan geçtiği zaman damgalarını kaydeder.
IP Datagramı Başlık Yapısı (Kontrol Toplamı - Checksum)

Checksum, IP başlığında taşıma sırasında bir hata olup olmadığını tespit etmek için kullanılır. Sadece başlık için hesaplanır (veri kısmı için değil). Her router'da TTL değiştiği için bu değer yeniden hesaplanır.

Gönderen Taraf (Hesaplama)

  1. Checksum alanı 0 olarak ayarlanır.
  2. Tüm başlık 16-bit'lik parçalara bölünür.
  3. Tüm bu parçalar toplanır.
  4. Toplamda taşan kısım (elde) varsa, toplamın sonuna eklenir.
  5. Çıkan sonucun 1'e tümleyeni (1's complement) alınır.
  6. Bu sonuç Checksum alanına yazılır.

Alıcı Taraf (Doğrulama)

  1. Gelen paketin başlığındaki tüm 16-bit'lik parçalar (Checksum dahil) toplanır.
  2. Elde (carry) varsa, toplama eklenir.
  3. Eğer başlıkta hata yoksa, bu toplamın 1'e tümleyeni 0 olmalıdır.

Hatasız bir başlıkta, doğrulama toplamının sonucu 1111 1111 1111 1111 (yani 1'e tümleyeni 0) olmalıdır.

5. IPv6 (Yeni Nesil IP)

IPv4 adres alanının tükenmesi sorununa çözüm olarak geliştirilmiştir.

  • 128 bit Adresleme: IPv4'ün 32 biti yerine 128 bit kullanarak astronomik sayıda (340 undesilyon) adres sağlar.
  • Geliştirilmiş Paket İşleme: IPv6 başlığı, IPv4'e göre daha basittir. Checksum ve Opsiyonlar gibi alanlar başlıktan çıkarılmıştır. Bu, router'ların paketleri daha hızlı işlemesini sağlar.
  • NAT İhtiyacını Ortadan Kaldırır: Adres alanı çok geniş olduğu için, her cihazın benzersiz bir genel IP adresine sahip olmasına olanak tanır ve NAT (Network Address Translation) ihtiyacını kaldırır.
  • Entegre Güvenlik: IPsec (IP Security) protokolü IPv6 standardına entegre edilmiştir.

IPv4'ten IPv6'ya Başlıktaki Değişiklikler

Kaldırılan Alanlar

  • IHL (HLEN)
  • Identification
  • Flags
  • Fragment Offset (Opsiyonel başlık oldu)
  • Header Checksum (Hata kontrolü üst katmanlara bırakıldı)
  • Options & Padding

Adı Değişen Alanlar

  • Service Type → Traffic Class
  • Total Length → Payload Length (Sadece veri uzunluğu)
  • Time to Live → Hop Limit
  • Protocol → Next Header

Yeni Eklenen Alan

  • Flow Label: Aynı akışa ait paketleri (örn: video akışı) etiketlemek ve QoS (Hizmet Kalitesi) sağlamak için kullanılır.

6. ICMP (Internet Control Message Protocol)

ICMP Nedir? (Genel Format)

ICMP, IP protokolünün ayrılmaz bir parçasıdır ve ağ katmanında çalışır. IP'nin "en iyi teslim" (best-effort) doğası gereği ortaya çıkan sorunları raporlamak ve ağ hakkında bilgi sorgulamak için kullanılır. ICMP mesajları, IP datagramının Payload kısmında (Protokol No: 1) taşınır.

Type (8 bits):     // Mesajın türü (örn: 8 = Echo Request)
Code (8 bits):     // Mesaj türünün alt nedeni (örn: Type 3, Code 1)
Checksum (16 bits): // Sadece ICMP mesajının bütünlüğü için
Rest of Header:  // Mesaj türüne göre değişir
Data Section:    // (Opsiyonel) Hata durumunda orijinal IP başlığı...

ICMP Hata Raporlama Mesajları (Detay)

Type 3: Destination Unreachable

Hedef Ağ (Code 0), Host (Code 1), Protokol (Code 2) veya Port (Code 3) ulaşılamaz olduğunda gönderilir. Ayrıca DF=1 iken parçalanma gerekirse (Code 4) bu hata kullanılır.

Type 11: Time Exceeded

Bir router paketin TTL değerini 0 yaptığında (Code 0 - traceroute bu mekanizmayı kullanır) veya hedef host parçalanmış paketin tüm parçalarını zamanında alamadığında (Code 1) gönderilir.

Type 5: Redirection

Bir router, bir host'un daha iyi bir rota kullanabileceğini fark ettiğinde (örn: aynı ağdaki başka bir router'a gitmesi gerekirken kendisine geldiyse) bu mesajı göndererek host'un yönlendirme tablosunu günceller.

ICMP Sorgu Mesajları (Detay)

Type 8 (Request) / Type 0 (Reply)

ping komutunun temelidir. Bir host'un veya router'ın ağda ulaşılabilir (hayatta) olup olmadığını test etmek için kullanılır. Identifier ve Sequence Number alanları, istek ve cevapları eşleştirmek için kullanılır.

Type 13 (Request) / Type 14 (Reply)

İki makine arasındaki gidiş-dönüş süresini (RTT) hesaplamak veya makinelerin saatlerini senkronize etmek için kullanılır. Original, Receive ve Transmit zaman damgalarını içerir.
