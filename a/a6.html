1. Soket Kavramı

Soket (Socket), TCP veya UDP kullanan iki bilgisayar arasındaki genel iletişim mekanizmasını sağlayan bir yapıdır. Ağ programlama için bir Standart API (Uygulama Programlama Arayüzü) olarak düşünülebilir.

Bir uygulamanın ağ üzerinden iletişim kurması için bir "kapı" veya "uç nokta" (endpoint) oluşturur. Soket, Uygulama Katmanı ile Taşıma Katmanı arasında bir arayüz görevi görür.

Soketi Oluşturan 3 Ana Bileşen:

  • IP Adresi: Cihazın ağdaki adresi.
  • Taşıma Katmanı Protokolü: İletişim türü (TCP veya UDP).
  • Port Numarası: Cihazdaki belirli bir uygulamayı hedefleyen kapı.

Akış Soketleri (Stream Sockets)

TCP temellidir. Güvenilir, sıralı, bağlantı yönelimli (connection-oriented) bir iletişim sağlar. Veri akışı kesintisizdir.

Datagram Soketler (Datagram Sockets)

UDP temellidir. Güvenilir olmayan, bağlantısız (connectionless) bir iletişim sağlar. Veri, paket (datagram) bazlı gönderilir.

Raw Soketler (Raw Sockets)

Ham paket yapısı kullanır. Taşıma katmanını atlayarak doğrudan Ağ katmanına (IP) erişir. Genellikle ping ve traceroute gibi analiz araçları için kullanılır.

2. İstemci-Sunucu Mimarisi

Aktif İstemci / Pasif Sunucu

Sunucu (Server)

  • Ağda pasif bir rol oynar.
  • Belirli bir port üzerinden bağlantı isteklerini bekler (listen).
  • Gelen isteklere cevap verir.
  • Bu yapıya pasif soket yapısı denir.

İstemci (Client)

  • Ağda aktif bir rol oynar.
  • Bağlantıyı başlatır (connect).
  • Sunucunun IP adresini ve port numarasını bilmesi gerekir.
  • Bu yapıya aktif soket yapısı denir.
Soketlerin Katmanlı Gösterimi (Descriptor Table)

İşletim sistemi, açılan her soketi bir "dosya tanımlayıcısı" (descriptor) gibi yönetir. Bir Tanımlayıcı Tablosu (Descriptor Table) tutulur. Bu tablo, bir soket tanımlayıcısını (örn: 3) o sokete ait dahili veri yapılarına bağlar.

Genel Soket Adres Yapısı (sockaddr)

Farklı protokol ailelerini destekleyebilen genel yapıdır.


struct sockaddr {
    unsigned short sa_family; // Adres ailesi (e.g. AF_INET)
    char           sa_data[14]; // Aileye özel adres bilgisi
};

TCP/IP (Internet) Adres Yapısı (sockaddr_in)

Uygulamada genellikle TCP/IP (IPv4) için özelleşmiş bu yapı kullanılır ve (struct sockaddr *) olarak dönüştürülür (cast edilir).


struct in_addr {
    unsigned long s_addr; // Internet adresi (32 bit)
};

struct sockaddr_in {
    unsigned short sin_family; // Internet protokolü (AF_INET)
    unsigned short sin_port;   // Adres portu (16 bit)
    struct in_addr sin_addr;   // Internet adresi (32 bit)
    char           sin_zero[8];  // Kullanılmaz (dolgu)
};
int sockid = socket(family, type, protocol);
  • family: İletişim domaini (örn: PF_INET - IPv4).
  • type: İletişim tipi (örn: SOCK_STREAM - TCP, SOCK_DGRAM - UDP).
  • protocol: Genellikle 0 (varsayılanı kullan) veya IPPROTO_TCP, IPPROTO_UDP.
int status = bind(sockid, &addrport, size); (Sunucu)

Soketi belirli bir IP/Port'a bağlar. Sunucular genellikle IP adresi olarak INADDR_ANY kullanır; bu, makinenin herhangi bir ağ arayüzünden (IP adresinden) gelen bağlantıları kabul etmesi anlamına gelir.

int status = listen(sockid, queueLimit); (TCP Sunucu)

Dinleme soketini (sockid) pasif moda geçirir. queueLimit (backlog), sunucu meşgulken kuyrukta bekleyecek maksimum istemci sayısını belirtir.

Bu "Listening socket" asla send() ve recv() için kullanılmaz, sadece accept() için kullanılır.

connect() (İstemci) ve accept() (Sunucu)

İstemci: connect(sockid, &foreignAddr, ...)

Sunucunun adresini (&foreignAddr) kullanarak bağlantı isteği gönderir. Sunucu accept() diyene kadar engellenir (bloke olur).

Sunucu: int s = accept(sockid, &clientAddr, ...)

Dinleyen sockid üzerinde bir bağlantı bekler. İstek geldiğinde, istemcinin adresini &clientAddr'a yazar ve yeni bir soket (s) döndürür. İletişim bu yeni soket üzerinden yapılır.

Veri Okuma/Yazma (TCP vs UDP)

TCP: send() / recv()

Bağlantı kurulduğu için hedef adres belirtmeye gerek yoktur. send(sockid, msg, ...) ve recv(sockid, buf, ...) kullanılır.

UDP: sendto() / recvfrom()

Bağlantı olmadığı için her göndermede hedef adres belirtilmelidir: sendto(sockid, msg, ..., &foreignAddr, ...). Her almada gönderenin adresi öğrenilir: recvfrom(sockid, buf, ..., &clientAddr, ...).

5. Örnek Kod Akışları (C Dili)

Sunucu (TCP) Kodu


// 1. Soket oluşturma
servSock = socket(PF_INET, SOCK_STREAM, IPPROTO_TCP);

// 2. Sokete port atama (bind)
addrport.sin_family = AF_INET;
addrport.sin_port = htons(servPort);
addrport.sin_addr.s_addr = htonl(INADDR_ANY);
bind(servSock, (struct sockaddr *) &addrport, ...);

// 3. Soketi dinleme
listen(servSock, MAXPENDING);

for (;;) { // 4. Tekrarla
    // a. Yeni bağlantı kabul etme (Bloke olur)
    clientSock = accept(servSock, ...);

    // b. İletişim (Echo)
    while (recv(clientSock, buffer, ...) > 0) {
        send(clientSock, buffer, ...);
    }

    // c. İletişim bitince istemci soketini kapat
    close(clientSock);
}

İstemci (TCP) Kodu


// 1. Soket oluşturma
clientSock = socket(PF_INET, SOCK_STREAM, IPPROTO_TCP);

// 2. Bağlantı kurma (connect)
addrport.sin_family = AF_INET;
addrport.sin_addr.s_addr = inet_addr(servIP);
addrport.sin_port = htons(servPort);
connect(clientSock, (struct sockaddr *) &addrport, ...);

// 3. İletişim (Gönder)
send(clientSock, echoString, ...);

// 3. İletişim (Al)
recv(clientSock, buffer, ...);

// 4. Bağlantıyı kapat
close(clientSock);

Sunucu (UDP) Kodu


// 1. Soket oluşturma
servSock = socket(PF_INET, SOCK_DGRAM, IPPROTO_UDP);

// 2. Sokete port atama (bind)
addrport.sin_family = AF_INET;
addrport.sin_port = htons(servPort);
addrport.sin_addr.s_addr = htonl(INADDR_ANY);
bind(servSock, (struct sockaddr *) &addrport, ...);

for (;;) { // 3. Tekrarla
    // a. İletişim (Al) (Bloke olur)
    // İstemci adresi (echoClientAddr) burada dolar
    recvMsgSize = recvfrom(servSock, buffer, ..., (struct sockaddr *) &echoClientAddr, ...);

    // a. İletişim (Gönder)
    // echoClientAddr adresine geri gönder
    sendto(servSock, buffer, recvMsgSize, ..., (struct sockaddr *) &echoClientAddr, ...);
}

İstemci (UDP) Kodu


// 1. Soket oluşturma
clientSock = socket(PF_INET, SOCK_DGRAM, IPPROTO_UDP);

// 2. Sunucu adresini hazırla
servAddr.sin_family = AF_INET;
servAddr.sin_addr.s_addr = inet_addr(servIP);
servAddr.sin_port = htons(servPort);

// 3. İletişim (Gönder)
// Hedef adres (servAddr) burada belirtilir
sendto(clientSock, echoString, ..., (struct sockaddr *) &servAddr, ...);

// 3. İletişim (Al)
// Sunucudan cevabı bekle
recvfrom(clientSock, buffer, ...);

// 4. Bağlantıyı kapat
close(clientSock);

7. Mesaj Oluşturma (Veri Kodlama ve Byte Dizilimi)

Ağ üzerinden sayılar gibi ham verileri gönderirken iki temel sorunla karşılaşırız. İlk sorun kodlamadır.

Çözüm 1: Karakter Kodlama (ASCII)

Sayıları metne (string) çevirip göndermek. (örn: sprintf(msgBuffer, "%d %d", x, y);)

Handikapları:
  • Ayırıcı (delimiter) gerekir.
  • Verimsizdir (örn: '1' rakamı 1 bayt yer kaplar).
  • Alıcı tarafında metni tekrar sayıya çevirmek (parsing) gerekir.

Çözüm 2: Değerleri (Binary) Gönderme

Sayıların bellekteki ham bayt karşılıklarını (örn: 4 bayt) doğrudan göndermek. (örn: send(sock, &x, sizeof(x));)

Gereklilik:

Her iki tarafın da bir protokol üzerinde anlaşması gerekir (int kaç bit? hangi kodlama?). Ancak bu, ikinci bir soruna yol açar: Byte Dizilimi.

Farklı makine mimarileri, tamsayıları (integer) bellekte farklı şekillerde sıralar.

Big-Endian (Yüksek Bayt Önce)

(örn: IBM, eski SPARC). MSB (En Yüksek Değerli Bayt) bellekte ilk adrese yazılır.

Sayı: 0x0A0B0C0D (A=MSB, D=LSB)

0A0B0C0D
AA+1A+2A+3

Little-Endian (Düşük Bayt Önce)

(örn: Intel x86, AMD64). LSB (En Düşük Değerli Bayt) bellekte ilk adrese yazılır.

Sayı: 0x0A0B0C0D (A=MSB, D=LSB)

0D0C0B0A
AA+1A+2A+3

Çözüm: Network Byte-Ordering (Ağ Bayt Sıralaması)

Ağ protokolleri tarafından kullanılan standart sıralama her zaman Big-Endian'dır. Veriler gönderilmeden önce Host sıralamasından Network sıralamasına dönüştürülmelidir. C fonksiyonları (htonl, htons) bu işi yapar.

Port numaraları (16 bit) ve IP adresleri (32 bit) gibi tamsayılar, ağa gönderilmeden önce Host bayt sıralamasından Network bayt sıralamasına dönüştürülmelidir.


// Host to Network (Göndermeden Önce)
u_long  htonl(u_long x);  // (Host to Network Long - 32 bit)
u_short htons(u_short x); // (Host to Network Short - 16 bit)

// Network to Host (Aldıktan Sonra)
u_long  ntohl(u_long x);  // (Network to Host Long - 32 bit)
u_short ntohs(u_short x); // (Network to Host Short - 16 bit)

Örnek (UDP İstemci ve Sunucu):


// --- İstemci (Gönderen) ---
unsigned short clientPort = 1111;
unsigned short message;

// Portu Network Byte Order'a çevir
message = htons(clientPort);

sendto(clientSock, &message, ...);


// --- Sunucu (Alıcı) ---
unsigned short clientPort;
unsigned short rcvBuffer;

recvfrom(servSock, &rcvBuffer, ...);

// Ağından gelen veriyi Host Byte Order'a çevir
clientPort = ntohs(rcvBuffer);

printf("Client's port: %d", clientPort);
